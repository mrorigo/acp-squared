ACP Native

ACP (Agent Communication Protocol) defines how agents talk with orchestration systems using HTTP-based commands like /runs and /runs/{id}/cancel. ZedACP follows the same semantics, but communicates via JSON-RPC over stdio instead of REST.

ACP² was built to bridge that gap — allowing existing IBM ACP clients to operate ZedACP agents without any code changes. It translates REST ↔ JSON-RPC in both directions, tracks sessions across restarts, and introduces a reliable persistence layer so developers can scale and debug distributed agents like first-class services.

Bring Zed Agents to the Cloud

ACP² allows Claude Code, Codex, and Gemini CLI to be deployed and operated as enterprise-grade cloud services. It turns local ZedACP-based tools into scalable, managed endpoints compatible with IBM ACP orchestration, enabling organizations to integrate advanced AI coding assistants directly into their cloud workflows.

By bridging the protocol divide, ACP² makes it possible for enterprises to expose these agents securely over REST, with session tracking, authentication, and persistence — unlocking real DevOps-level control over interactive agents that used to only run locally.
